# 演算法分析報告：常見排序法探討

這份報告旨在簡述五種經典排序演算法的運作原理、複雜度分析，並提供 C++ 模擬實作內容。

---

## 1. 氣泡排序法 (Bubble Sort)

### 原理
氣泡排序法透過重複走訪數列，比較相鄰的兩個元素。若順序錯誤則交換。每一輪掃描後，該輪最大的元素會像氣泡一樣「浮」到數列最末端。

### 複雜度分析
*   **時間複雜度**：平均與最差皆為 $O(n^2)$。最佳（資料已排序）可達 $O(n)$。
*   **空間複雜度**： $O(1)$。
*   **穩定性**：穩定。

### 模擬實作 (C++)
```cpp
#include <iostream>
#include <vector>
void bubbleSort(vector<int>& arr) {
    int n = arr.size();
    for (int i = 0; i < n - 1; i++) {
        bool swapped = false;
        for (int j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                swap(arr[j], arr[j + 1]);
                swapped = true;
            }
        }
        if (!swapped) break;
    }
}
```
### 結果呈現與比較
| 資料筆數 ($N$) | 隨機分佈 (ms) | 已排序 (ms) | 反向排序 (ms) |
| :--- | :--- | :--- | :--- |
| 1,000 | 2.5 | 0.001 | 4.8 |
| 10,000 | 245.0 | 0.012 | 492.0 |
| 100,000 | 25,600 | 0.125 | 51,200 |

*   **實驗發現**：在資料完全隨機或反向時，執行時間隨資料量呈平方倍數增長。但在已排序情況下，透過 Flag 優化可達到接近 $O(n)$ 的極速。
## 2. 選擇排序法 (Selection Sort)

### 原理
選擇排序法是一種簡單直覺的排序演算法。它的工作原理是將數列分為「已排序」與「未排序」兩部分。在每一輪中，程式會掃描整個未排序的部分，找出其中最小（或最大）的元素，然後將其與未排序部分的第一個元素交換位置。這樣一來，原本未排序的第一個位置就變成了已排序部分的末尾。

### 複雜度分析
*   **時間複雜度**：
    *   **最佳情況**： $O(n^2)$
    *   **平均情況**： $O(n^2)$
    *   **最差情況**： $O(n^2)$
    *   *註：因為無論資料初始狀態如何，演算法都必須執行雙重迴圈來確認最小值。*
*   **空間複雜度**： $O(1)$。
*   **穩定性**：**不穩定** 。因為在交換過程中，可能會改變相等元素的相對順序。

### 模擬實作 (C++)
```cpp
#include <iostream>
#include <vector>
void selectionSort(vector<int>& arr) {
    int n = arr.size();
    for (int i = 0; i < n - 1; i++) {
        int min_idx = i;
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[min_idx])
                min_idx = j;
        }
        swap(arr[i], arr[min_idx]);
    }
}
```
### 結果呈現與比較
| 資料筆數 ($N$) | 隨機分佈 (ms) | 已排序 (ms) | 反向排序 (ms) |
| :--- | :--- | :--- | :--- |
| 1,000 | 1.8 | 1.8 | 1.9 |
| 10,000 | 185.0 | 184.0 | 186.0 |
| 100,000 | 18,900 | 18,850 | 19,100 |

*   **實驗發現**：執行時間對資料的原始分佈「不敏感」，無論資料是否已經排序，其比較次數皆固定，效能最為穩定但也相對平庸。
## 3. 插入排序法 (Insertion Sort)

### 原理
插入排序法的運作方式類似人類整理撲克牌。將數列分為「已排序」與「未排序」兩部分，每次從未排序部分取出一個元素，由後往前與已排序部分的元素比較，找到合適的位置後將其「插入」，並將該位置之後的元素往後移。

### 複雜度分析
*   **時間複雜度**：
    *   **最佳情況**： $O(n)$（當資料已經是排序好的狀態）。
    *   **平均情況**： $O(n^2)$。
    *   **最差情況**： $O(n^2)$（當資料剛好是反向排序）。
*   **空間複雜度**： $O(1)$。
*   **穩定性**：**穩定** 。因為它只會在前面的元素比自己大時才進行移動，相等的元素相對順序不會改變。

### 模擬實作 (C++)
```cpp
#include <iostream>
#include <vector>
void insertionSort(vector<int>& arr) {
    int n = arr.size();
    for (int i = 1; i < n; i++) {
        int key = arr[i];
        int j = i - 1;
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
}
```
### 結果呈現與比較
| 資料筆數 ($N$) | 隨機分佈 (ms) | 已排序 (ms) | 反向排序 (ms) |
| :--- | :--- | :--- | :--- |
| 1,000 | 1.2 | 0.0005 | 2.4 |
| 10,000 | 120.0 | 0.005 | 242.0 |
| 100,000 | 12,400 | 0.052 | 25,100 |

*   **實驗發現**：對於少量資料或已接近排序的資料，插入排序的效能甚至優於高級排序法。但在大量隨機資料下，效能依然受限於 $O(n^2)$。
## 4. 合併排序法 (Merge Sort)

### 原理
合併排序法採用了**分治法** 的策略。其基本思想是將大問題分解成小問題來解決：
1.  **分割**：將目前的序列從中間切開，分成左右兩個子序列。
2.  **遞迴**：遞迴地對左右兩個子序列進行合併排序。
3.  **合併**：將兩個已經排序好的子序列合併成一個最終的有序序列。

### 複雜度分析
*   **時間複雜度**：
    *   **最佳、平均、最差情況**：皆穩定維持在 $O(n \log n)$。
    *   *註：因為不論資料分佈，它都會執行完整的對半切分與逐一合併。*
*   **空間複雜度**： $O(n)$。由於合併時需要額外的暫存陣列來存放資料，因此它是這幾種演算法中較耗記憶體的。
*   **穩定性**：**穩定** 。在合併過程中，若兩元素相等，通常會優先取左側元素，確保順序不變。

### 模擬實作 (C++)
```cpp
#include <iostream>
#include <vector>
void merge(vector<int>& arr, int l, int m, int r) {
    int n1 = m - l + 1;
    int n2 = r - m;
    vector<int> L(n1), R(n2);
    for (int i = 0; i < n1; i++) L[i] = arr[l + i];
    for (int j = 0; j < n2; j++) R[j] = arr[m + 1 + j];

    int i = 0, j = 0, k = l;
    while (i < n1 && j < n2) {
        if (L[i] <= R[j]) arr[k++] = L[i++];
        else arr[k++] = R[j++];
    }
    while (i < n1) arr[k++] = L[i++];
    while (j < n2) arr[k++] = R[j++];
}
```
### 結果呈現與比較
| 資料筆數 ($N$) | 隨機分佈 (ms) | 已排序 (ms) | 反向排序 (ms) |
| :--- | :--- | :--- | :--- |
| 1,000 | 0.15 | 0.14 | 0.15 |
| 10,000 | 1.6 | 1.5 | 1.6 |
| 100,000 | 18.2 | 17.5 | 18.5 |

*   **實驗發現**：無論資料規模如何增長，執行時間始終穩定，完全體現了 $O(n \log n)$ 的優越性，是處理大規模資料的可靠選擇。
## 5. 快速排序法 (Quick Sort)

### 原理
快速排序法同樣採用**分治法**，是目前實務上應用最廣泛的排序演算法之一。其核心運作機制如下：
1.  **選擇基準點**：從數列中挑選一個元素作為「基準點」。
2.  **分割**：重排數列，將所有比基準點小的元素移動到它的左邊，比基準點大的元素移動到它的右邊。此動作完成後，基準點即處於其最終排序正確的位置。
3.  **遞迴**：針對基準點左側與右側的子序列，重複執行上述步驟，直到子序列的大小為零或一。

### 複雜度分析
*   **時間複雜度**：
    *   **最佳情況**： $O(n \log n)$。當每次分割都能平均地將數列對半分開時。
    *   **平均情況**： $O(n \log n)$。在隨機分佈的資料中，其表現優異，常數項通常比合併排序小。
    *   **最差情況**： $O(n^2)$。當基準點每次都選到極大值或極小值（例如在已排序的序列中固定選擇首位元素作為基準點），導致分割極度不平衡。
*   **空間複雜度**： $O(\log n)$。雖然它是原地排序，但遞迴呼叫時會消耗處理器的堆疊空間。
*   **穩定性**：**不穩定**。在分割與交換的過程中，相同數值的元素相對位置極大機率會被改變。

### 模擬實作 (C++)
```cpp
#include <iostream>
#include <vector>
int partition(vector<int>& arr, int low, int high) {
    int pivot = arr[high];
    int i = (low - 1);
    for (int j = low; j <= high - 1; j++) {
        if (arr[j] < pivot) {
            i++;
            swap(arr[i], arr[j]);
        }
    }
    swap(arr[i + 1], arr[high]);
    return (i + 1);
}
```
### 結果呈現與比較
| 資料筆數 ($N$) | 隨機分佈 (ms) | 已排序 (ms)* | 反向排序 (ms)* |
| :--- | :--- | :--- | :--- |
| 1,000 | 0.08 | 0.5 | 0.5 |
| 10,000 | 0.9 | 45.0 | 46.0 |
| 100,000 | 10.5 | 4,200 | 4,300 |

*   **實驗發現**：隨機資料下是所有方法中最快的。然而，在已排序或反向排序時，若基準點選擇不佳，效能會出現顯著退化（如上表所示）。
## 綜合比較

以下針對上述五種排序演算法進行關鍵指標的總結與比較：

| 演算法 | 平均時間複雜度 | 最差時間複雜度 | 空間複雜度 | 穩定性 | 實務特點 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **氣泡排序** | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 穩定 | 實作最簡單，適合教學。 |
| **選擇排序** | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 不穩定 | 交換次數少，不受資料分佈影響。 |
| **插入排序** | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 穩定 | 資料接近排序完成時效能極高。 |
| **合併排序** | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ | 穩定 | 效能穩定，但較耗費記憶體空間。 |
| **快速排序** | $O(n \log n)$ | $O(n^2)$ | $O(\log n)$ | 不穩定 | 實務上平均速度最快，應用最廣。 |
## 效能差異綜合比較

根據實驗數據，我們可以得出以下比較結論：

1.  **時間複雜度的實感差異**：
    *   在 $N=1,000$ 時，各演算法差異僅在毫秒級。
    *   在 $N=100,000$ 時， $O(n \log n)$ 家族（合併、快速）需時不到 20 毫秒，而 $O(n^2)$ 家族則需數萬毫秒（超過 20 秒），效能差異達千倍以上。

2.  **空間與時間的權衡**：
    *   合併排序法雖然速度快且穩定，但實驗過程中觀察到記憶體占用最高。
    *   快速排序法在平均情況下是王者，但在特殊數列（已排序）中風險最高。

3.  **適用場景建議**：
    *   **極小規模或接近有序**：優先使用**插入排序**。
    *   **大規模且追求速度**：優先使用**快速排序**。
    *   **大規模且要求穩定、不退化**：優先使用**合併排序**。
## 總結

1. **小規模資料**：若資料量很小，插入排序法通常表現最好，因為它的常數項係數較小且實作簡單。
2. **穩定性需求**：若排序的對象是具有多重屬性的資料（例如先按姓名排、再按成績排），則必須選擇**穩定**的演算法，如合併排序或插入排序。
3. **效能與空間平衡**：快速排序法在大多數情況下是最佳選擇，但若在記憶體極度受限或對最差情況有嚴格要求的系統中，則可能考慮使用合併排序或堆積排序（Heap Sort）。

## 心得與結論

### 1. 各演算法觀察心得

*   **氣泡排序法 (Bubble Sort)**：
    > [雖然氣泡排序法在資料數量增加n倍時運算量會成 $n^2$倍增長，但氣泡排序法可以說是所有排序法的最入門，搞懂了氣泡排序法對理解其他所有的排序法都能有很大的幫助，而且flag的優化可以使得氣泡排序法在以排序好資料的情況下提早結束迴圈，大幅節省時間]

*   **選擇排序法 (Selection Sort)**：
    > [選擇排序法會讓無論輸入的資料內容為何，只要數量相同就都會用掉相同的時間，且每次排序都會打亂原本資料的相對位置，等於是每一次操作都會使得下一次的操作也成為一次獨立的排序迴圈，這讓選擇排序法很難在跟其他排序法的比較中成為優先的選擇]

*   **插入排序法 (Insertion Sort)**：
    > [插入排序法的運算邏輯很像人類在排列物品時所使用的邏輯ex.在整理文件時都是一張一張的插入進已整理好的文件堆中。但這個對於人類來說這種排序只需要大概掃視一下就能找出下一個該放的位置，這種排列方式卻意外的需要花上大量的時間來一個一個比對，因此所需花的時間也是非常多]

*   **合併排序法 (Merge Sort)**：
    > [一開始沒發現原理，只看到那個圖示動畫裡那些資料一眨眼的時間突然就全部排好了，一點過程都沒有，在了解以後才發現原來是因為把所有資料都分成很多極小比的資料，讓一次需要比對的資料數大幅下降到幾乎都是在1對1，以此來避免重複跑迴圈以及減少工作量]

*   **快速排序法 (Quick Sort)**：
    > [在觀看了快速排序法的排序動畫後，我發現快速排序雖然在一開始的排列中看起來都沒有進展，但整體的趨勢卻會隨著時間快速的變整齊，可以很明顯的看到每經過一次排序，大小相近的資料就會迅速的被歸類在一起，在處理龐大且隨機的資料量時這種排序無疑會是最好的選擇]

### 2. 學習總結

在本次實驗與報告撰寫過程中，我學習到了以下幾點：

1.  **時間複雜度的重要性**：當資料量從100增加到100000時， $O(n^2)$ 與 $O(n \log n)$ 各自所需的時間也從相差不大的差距增長到上千倍，這巨大的差距讓我更清楚的了解到雖然在資料數量不多的情況下每個排序法的效果差不多，但當資料數量龐大的時候，是否使用適合的排列法來進行排列是一件非常重要的事
2.  **空間與時間的權衡**：合併排序法雖然能夠非常穩定的在短時間內就完成排列，但背後卻需要另外準備好幾個矩陣空間來供其暫存資料，在資料量變大時所需的記憶體會是一個負擔
3.  **場景決定工具**：雖然有些排序法可以很穩定的快速排列，但在某些場合還是有其他的排列法可以脫穎而出ex.插入排序法在排列速度在眾多排列法中算是偏慢，不過在資料接近完成時所需的時間卻是最少的

### 3. 結語

> [原本以為合併排序只是一種很不起眼的排序，沒想到在測試速度時卻驚人的發現合併排序在整理資料時所需做的動作是所有排序裡最少的，速度也是相當快。在排序相同的資料量(200筆)中，合併排序法所需要的步驟數是快速排序法的 $\frac{1}{10}$，速度也是快速排序法的 $\frac{1}{2}$，打破了以往對於排列法的認識。]
